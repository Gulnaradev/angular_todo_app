# Angular Todo App — QA Test Assignment

Данный репозиторий содержит результаты тестирования Angular Todo App в рамках тестового задания на позицию QA Engineer.

Работа включает в себя:
- анализ требований и особенностей приложения,
- тестовую документацию,
- E2E автотесты на Playwright (TypeScript),
- выявленный дефект и его описание,
- отчёт о проделанной работе и стратегию QA.

---

## Цель работы

Целью задания являлась организация процесса тестирования веб-приложения:
- проектирование тестовой документации,
- покрытие ключевых пользовательских сценариев,
- проверка граничных и edge-case сценариев,
- демонстрация подходов к автоматизации и QA-процессам.

---

## Используемые инструменты и технологии

- Playwright
- TypeScript
- Page Object Pattern
- Markdown (тестовая документация)
- JSONPlaceholder API (внешний тестовый API)

---

## Требования для запуска

- Node.js v18+
- npm v9+
- Playwright v1.40+
- Поддерживаемые браузеры: Chromium, Firefox, WebKit

---

## Установка и запуск приложения

```bash
npm install
npm start
````

После запуска приложение доступно локально [http://localhost:4200](http://localhost:3000).

---

## Запуск автотестов

Запуск всех тестов:

```bash
npx playwright test
```

Запуск тестов в Chromium:

```bash
npx playwright test --project=chromium
```

Запуск с открытым браузером:

```bash
npx playwright test --project=chromium --headed
```

Просмотр HTML-отчёта:

```bash
npx playwright show-report
```

## Покрытие автотестами

### CRUD (основной функционал)

* создание задачи
* изменение статуса задачи
* редактирование задачи
* удаление задачи

### Фильтрация

* All
* Active
* Completed

### Edge / Corner cases

* данные не сохраняются после перезагрузки страницы (ограничение JSONPlaceholder API)
* невозможность создания задачи с пустым значением или пробелами

---

## Обнаруженные дефекты

В ходе тестирования был выявлен дефект:

* функция редактирования задачи не сохраняет изменённое имя после нажатия кнопки Save

Дефект оформлен в виде баг-репорта в файле:

```
docs/BUG_REPORTS.md
```

---

## Особенности внешнего API

Приложение использует публичный JSONPlaceholder API со следующими ограничениями:

* изменения не сохраняются после обновления страницы;
* возможная задержка ответа до 2 секунд.

Данные особенности учтены и покрыты отдельными edge-case тестами.

---

## Подход к тестированию

* тесты сгруппированы по логике (CRUD, Filters + Edge cases + Input validation);
* используется Page Object Pattern для поддерживаемости кода;
* применяются асинхронные ожидания без жёстких таймаутов;
* тестовые данные генерируются динамически;
* тест-кейсы дополнительно описаны в формате Gherkin (BDD).
