# TEST CASES — Angular Todo App

Формат: таблица.
Обоснование формата: для небольшого объёма удобнее таблица — быстро читать/ревьюить, можно легко переносить в TestRail/Jira.

| ID | Название | Предусловия | Шаги | Ожидаемый результат |
|---|---|---|---|---|
| TC-01 | Загрузка приложения | App запущен | Открыть главную страницу | Видны поле ввода новой задачи и список задач |
| TC-02 | Создание задачи (Enter) | - | Ввести текст в поле → Enter | Задача появляется в списке |
| TC-03 | Создание задачи с длинным названием | - | Ввести 200+ символов → Enter | Приложение не падает, задача создаётся/отображается корректно (возможно с переносом/обрезкой) |
| TC-04 | Создание задачи с пустым вводом | - | Оставить поле пустым → Enter | Задача НЕ создаётся |
| TC-05 | Создание задачи из пробелов | - | Ввести "   " → Enter | Задача НЕ создаётся (или trim → пусто → не создаётся) |
| TC-06 | Отметка задачи выполненной | Есть активная задача | Нажать checkbox | Checkbox отмечен, задача считается completed |
| TC-07 | Снятие отметки выполненной | Есть completed задача | Нажать checkbox | Checkbox снят, задача active |
| TC-08 | Редактирование задачи | Есть задача | Нажать Edit → изменить текст → Save | Название меняется, edit-mode закрывается |
| TC-09 | Редактирование в пустую строку | Есть задача | Edit → очистить поле → Save | Ожидаемое поведение: запретить/не сохранять пустое значение или показать ошибку (зафиксировать требование) |
| TC-10 | Удаление задачи | Есть задача | Нажать Delete | Задача исчезает из списка |
| TC-11 | Фильтр Active | Есть active и completed | Нажать Active | Видны только active |
| TC-12 | Фильтр Completed | Есть active и completed | Нажать Completed | Видны только completed |
| TC-13 | Фильтр All | Есть active и completed | Нажать All | Видны все задачи |
| TC-14 | Edge: неперсистентность после refresh | Создать новую задачу | Создать → обновить страницу | Созданная задача пропадает (ожидаемо из-за JSONPlaceholder) |
| TC-15 | Edge: задержка API до 2с | Сеть/мок | Открыть страницу при замедленном ответе | UI не “ломается”, после ответа список отображается |
